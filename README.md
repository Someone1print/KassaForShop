# Система контроля кассовых операций магазина

Полнофункциональное веб-приложение для управления кассовыми операциями магазина на Django 5+.

## Возможности

- ✅ Учёт продаж и возвратов товаров
- ✅ Контроль кассовых смен
- ✅ Управление товарами и остатками
- ✅ Отчёты по выручке и кассирам
- ✅ История операций и поиск
- ✅ Многопользовательская работа (админ и кассир)
- ✅ API для работы кассы (штрихкод, оформление покупок)
- ✅ Современный UI с Bootstrap 5

## Технологии

- Python 3.12+
- Django 5+
- Django REST Framework
- SQLite (по умолчанию) / PostgreSQL (через .env)
- Bootstrap 5
- Chart.js (для графиков)

## Установка

### 1. Клонирование и установка зависимостей

```bash
# Создайте виртуальное окружение
python -m venv venv

# Активируйте виртуальное окружение
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt
```

### 2. Настройка базы данных

По умолчанию используется SQLite. Для использования PostgreSQL создайте файл `.env` в корне проекта:

```env
DB_ENGINE=postgresql
DB_NAME=kassa_db
DB_USER=your_user
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
SECRET_KEY=your-secret-key-here
DEBUG=True
```

### 3. Миграции

```bash
python manage.py migrate
```

### 4. Создание демо-данных

```bash
python manage.py seed_demo_data
```

Это создаст:
- **Администратор**: `admin` / `admin12345`
- **Кассир**: `cashier1` / `cashier12345`
- **10 товаров** с штрихкодами и остатками

### 5. Запуск сервера

```bash
python manage.py runserver
```

Откройте браузер: http://127.0.0.1:8000/

## Роли пользователей

### Администратор

- Управление товарами (CRUD)
- Управление кассирами (CRUD, блокировка/активация)
- Просмотр всех смен и отчётов
- Генерация отчётов

**Вход**: `admin` / `admin12345`

### Кассир

- Открытие/закрытие смены
- Регистрация продаж и возвратов
- Просмотр своих продаж и смен

**Вход**: `cashier1` / `cashier12345`

## Основные страницы

### Для кассира

- **Dashboard** (`/`) - главная страница со статистикой
- **Касса** (`/pos/`) - кассовый экран для продаж
  - Ввод штрихкода (Enter для добавления в корзину)
  - Поиск по названию товара
  - Режим возврата
  - Выбор способа оплаты (наличные/карта)
- **Смены** (`/shifts/`) - список своих смен
- **Товары** (`/products/`) - просмотр товаров

### Для администратора

- **Dashboard** (`/`) - статистика и графики
- **Смены** (`/shifts/`) - все смены всех кассиров
- **Товары** (`/products/`) - управление товарами
- **Отчёты** (`/reports/`) - отчёты по продажам и кассирам
- **Кассиры** (`/admin-panel/cashiers/`) - управление кассирами
  - Создание/редактирование кассиров
  - Просмотр смен каждого кассира
  - Блокировка/активация

## API Endpoints

### Товары

- `GET /api/products/` - список товаров (с поиском: `?search=...`)
- `GET /api/products/by-barcode/<barcode>/` - товар по штрихкоду

### Касса

- `POST /api/pos/checkout/` - оформление покупки/возврата
  ```json
  {
    "receipt_type": "SALE" | "RETURN",
    "payment_method": "CASH" | "CARD",
    "items": [{"product_id": 1, "qty": 2}, ...]
  }
  ```

### Смены

- `GET /api/shifts/` - список смен
- `POST /api/shifts/open/` - открыть смену
- `POST /api/shifts/<id>/close/` - закрыть смену

### Чеки

- `GET /api/receipts/<id>/` - детали чека

## Модели данных

- **Product** - товар (название, штрихкод, цена, остаток)
- **Shift** - кассовая смена (кассир, время открытия/закрытия, статус)
- **Receipt** - чек (продажа/возврат, способ оплаты, сумма)
- **ReceiptItem** - позиция в чеке (товар, количество, цена)
- **AuditLog** - журнал аудита операций

## Бизнес-логика

- Продажа возможна только при открытой смене
- Проверка остатков товара перед продажей
- Автоматическое обновление остатков при продаже/возврате
- Смена становится неизменяемой после закрытия
- Ведение журнала аудита всех операций
